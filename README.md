# Unity Portfolio

## 조작 방법

* WASD 키를 이용하여 플레이어를 이동합니다.
* 마우스 클릭 방향으로 플레이어가 회전합니다.
* 마우스 좌클릭으로 공격합니다.
* Input System을 사용하여 입력을 처리하였습니다.

---

## 구현 기능

### 1-1. 플레이어 이동

* Input System 기반 이동 구현
* Vector3 정규화 적용
* Quaternion.Slerp를 이용한 자연스러운 회전 처리

---

### 1-2. 플레이어 공격

* 마우스 위치를 Raycast로 월드 좌표로 변환
* 마우스 클릭 방향으로 플레이어 회전
* 공격 쿨타임 적용
* 부채꼴 범위 공격 구현

#### 공격 판정

* 공격 범위 내 대상 검사
* Vector3.Dot을 이용한 각도 판정
* 거리 + 각도 조건을 모두 만족하는 몬스터에게 피해 적용

---

### 2. 몬스터 감지 및 회전

* Vector3.Dot을 이용한 시야각 판정
* sqrMagnitude를 이용한 거리 계산
* Quaternion.RotateTowards를 이용한 회전 구현
* 플레이어 감지 및 추적 기능 구현

---

### 3-1. 거리 기반 상태 판정

* sqrMagnitude를 사용하여 거리 계산
* 공격 거리와 추적 거리를 구분하여 상태 전이 구현
* 거리 조건에 따라 Idle, Chase, Attack 상태 전이 구현

---

### 3-2. 플레이어 공격 판정

* 공격 범위 내 몬스터 검사
* 거리와 시야각을 동시에 검사
* 부채꼴 범위 공격 구현

---

### 4-1. 자료구조 활용

* List를 사용하여 몬스터를 관리
* 몬스터 순회 및 상태 확인
* 사망한 몬스터 제거 기능 구현

### 4-2. 몬스터 매니저

* List 기반 몬스터 관리

* 남은 몬스터 수 확인 기능 구현

---
#### List 선택 이유

현재 프로젝트는 몬스터를 순차적으로 탐색하고 범위 공격 판정을 수행하는 것이 주 목적이므로 List를 선택하였습니다.

Dictionary는 특정 데이터를 빠르게 검색하는 데 유리하지만, 현재 구현에서는 모든 몬스터를 순회하며 거리 및 각도 판정을 수행하는 구조이므로 List가 더 적합하다고 판단하였습니다.
